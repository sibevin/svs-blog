log aggregation - fluentd + elasticsearch + kibana - Ash@hSATAC

  • log files across servers, grep, autoscaling -> lost log, keep
  • fluentd - data collector, simple, flexible
  • fluentd instace is a node(server, client or server+client)
  • dataflow - input/output
  • input - in_forward, in_http, in_tail, in_exec, in_syslog
  • output - out_file, out_foward, out_exec, out_copy(fork two ways), out_stdout, out_s3
  • buffered output - delay over a period of time or over a fixedd size, for large traffic
  • time sliced - daliy, hourly, ...
  • elasticsearch - distributed restful search and analytics
  • kibana - es frontend, realtime query frontend, data visualizer
  • kiban - query & filter, panels(charts)
  • rails & fluentd integration
  • official doc - rails = lograge gem + act-fluent-logger-rails, fluentld = fluent-plugin-parser + fluent-plugin-elasticsearch
  • x: lograge is hidding logs, act-fluent-logger-rails crasheds puma, JSON encode / decode
  • solution 2 - act-fluent-logger-rails + unicore
  • x: log is not search friendly, unicorn return 404 when massive requests
  • logging should not effect the application
  • solution 3 - don't use act-fluent-logger-rails, keep lograge, using logstash to format, logstash-logger to converts other logs to logstash format, output to production.log
  • fluentd - tail production.log with format jsson, forward to staticon fluentd
